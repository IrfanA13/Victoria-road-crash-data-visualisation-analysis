# Victoria Road Crash Data Visualisation Analysis
## Project Overview
This project analyses real-world road crash data from Victoria, Australia, to uncover patterns in crash trends, severity, geographic distribution, and vehicle-related risk factors.

Using Python for initial data cleaning and exploratory analysis, and R for visualisation, the project transforms raw datasets into meaningful insights that support evidence-based decision making in road safety.

The analysis simulates a real-world data analyst workflow, focusing on both technical execution and clear communication of insights.

# Analytical Approach
The project began with data preparation and cleaning using Python, including handling missing values, standardising variables, and validating dataset quality. This ensured a reliable foundation before moving into analysis.

The cleaned data was then used in R to develop a series of visualisations, each designed to explore a different aspect of the dataset. The approach combines exploratory data analysis with visual storytelling to identify meaningful patterns and trends.

Each visualisation was supported by interpretation and reasoning, linking the findings to real-world factors such as traffic patterns, seasonal behaviour, and vehicle characteristics.

# Key Insights

The analysis revealed clear seasonal trends in crash frequency, with higher incidents during warmer months driven by increased travel and mobility.

Geographic analysis showed that urban areas, particularly Melbourne CBD and surrounding suburbs, experience the highest crash density due to traffic volume and complex road conditions.

In contrast, rural regions recorded fewer crashes but significantly higher fatality rates, likely due to higher speeds and longer emergency response times.

Vehicle-related analysis indicated that newer vehicles generally reduce crash severity due to improved safety features, while motorcycles consistently show higher risk due to lack of physical protection.

# Visualisations Included

This project includes five key visualisations:

Time-series analysis of crash trends and seasonal patterns
Geographic heatmap of crash density across Victoria
Crash severity distribution across urban and rural regions
Impact of vehicle type and manufacturing period on crash severity
Distribution of vehicle damage across major car brands

Each visualisation is designed to provide a unique perspective and is supported by detailed interpretation. The individual corresponding R files are also available along with the Report 

# Tools and Technologies
This project was completed using Python for data cleaning and exploratory data analysis, and R for developing visualisations. Key R packages included tidyverse for data manipulation, ggplot2 for visualisation, and lubridate for handling date and time variables.

This project demonstrates the ability to work with real-world datasets, perform data cleaning and analysis, and communicate insights effectively through visualisation.

It highlights a strong combination of technical skills and business understanding, showing how data can be used to support decision-making and improve real-world outcomes such as road safety.
